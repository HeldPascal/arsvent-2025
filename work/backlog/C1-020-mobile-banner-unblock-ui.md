## Goal
Ensure the mobile devices banner never blocks core interactions.

## Scope
- Adjust banner layout/behavior on small screens
- Add dismiss or auto-collapse behavior

## Functional Requirements
- Banner must not cover primary interactions on mobile viewports
- Banner can be dismissed and stays dismissed for the session
- Behavior is consistent across common mobile browsers

## Acceptance Criteria
- [ ] Banner never blocks taps for core actions on mobile
- [ ] Banner can be dismissed and stays dismissed for the session
